# Add Scale Lengths k=9 and k=10

## Motivation

Scale lengths k=9 and k=10 are the complements of k=3 and k=2 respectively. A k-note scale's complement (flipping all 12 bits) yields a (12-k)-note scale. Since complement commutes with rotation, the shape graph topology is identical ŌĆö same number of shapes, same edges. Only the bitmasks and musical names differ.

Deriving k=9 from k=3 (and k=10 from k=2) rather than computing from scratch preserves shape index correspondence, which enables reusing manual graph layouts in the future.

## Script Changes (`scripts/src/main.ts`)

### New function: `deriveComplementData`

```typescript
function deriveComplementData(source: ScaleLengthData, newK: number): ScaleLengthData
```

**Shapes**: For each source shape (preserving array index):
- `compCanon = canonical(complement(source.canonical))`
- `offset = findRotation(compCanon, complement(source.canonical))`
- `displayBitmask = displayRotation(compCanon)`
- `symmetryOrder` copied directly
- Mode rotations: `(mode.rotation + offset) % 12`

**Scales**: For each source scale (preserving array index):
- `newBitmask = complement(scale.bitmask)`
- `pitchClasses = pitchClassesFromBitmask(newBitmask)`
- `shapeIndex` copied directly
- `rotation = (scale.rotation + shapeOffset) % 12`
- `names = getScaleNames(newBitmask)`

**Edges**: `shapeEdges` and `scaleEdges` copied as-is (index-based, 1:1 correspondence).

**Mode names**: Re-filled from scale names using the same logic as `computeScaleLengthData` step 6.

### Rotation derivation

The complement rotation is `(r + offset) % 12` where offset is a per-shape constant:

If `scale = rotate(canon, r)`, then:
- `complement(scale) = rotate(complement(canon), r)`
- `complement(canon) = rotate(compCanon, offset)`
- Therefore `complement(scale) = rotate(compCanon, (r + offset) % 12)`

This was verified empirically across all 19 k=3 shapes.

### Generation order

```typescript
for (const k of [2, 3, 4, 5, 6, 7, 8]) {
  // compute and write as before
}
// Derive complements from already-computed data
const k9data = deriveComplementData(k3data, 9);
const k10data = deriveComplementData(k2data, 10);
// Write k9.json, k10.json
```

The k=2 and k=3 data objects need to be retained after computation so they can be passed to `deriveComplementData`.
